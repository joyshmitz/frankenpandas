===================================================================================

Fourth in the eydcr Scalar-bound-elementwise-ops family (after diff 30x,
pct_change 38x, combine_first 5.6x). ffill/bfill materialized Vec<Scalar>,
carried a &Scalar last/next-valid, cloned per element, and rebuilt via

LEVER (one — typed nullable carry-fill, both directions): read raw f64 + validity
via Column::as_f64_slice_with_validity, carry the last (ffill) / next (bfill)
valid value into a typed output buffer + validity mask, emit via
from_f64_values_with_validity. DataFrame.ffill/bfill delegate per-column through
apply_all_columns, so they benefit transitively.

ISOMORPHISM (per-row carry — no f64 arithmetic, no reordering):
  - valid row: keeps its value (present Float64), same as the general path.
  - missing row within `limit` consecutive fills: takes the carried valid f64
    (a present Float64) — the carry value equals the original `&Scalar`'s f64.
  - missing row past `limit`, or before any valid value: stays missing — its
    cleared validity bit + stored datum materialises Null(NaN) (non-NaN datum)
    or Float64(NaN) (NaN datum), exactly the original `val.clone()` of the
    missing slot under the LazyNullableFloat64 convention.
  - `consecutive_fills` increments before the limit check and resets on a valid
    value — identical to the general path.
