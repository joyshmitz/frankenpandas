br-frankenpandas-yiqv5 — Deferred null-introducing Utf8 gather (lazy join value lanes)
======================================================================================

LEVER (one): reindex_by_positions' all-valid-Eager Utf8 null path now returns a
new LazyGatherUtf8 backing instead of eagerly gathering bytes. It shares the
source Scalar Arc (O(1), post-cfr08 Eager is Arc<[Scalar]>) and captures only
the position plan (usize::MAX == missing) + an eagerly-built validity bitset
(O(output) bits, cheap). The gathered string Scalars materialize once on demand
— so a null-introducing string join whose value lanes are only counted
(index/shape consulted, bytes never read) skips the O(output) byte memmove +
per-column buffer alloc/offsets/drop entirely. Mirrors the inner-join lazy-lane
win (ddb4c8e9) for the cmxjz Utf8 gather path. Contiguous / nullable sources
keep the existing eager cmxjz path; non-Utf8 keep their typed fast paths.

WHY str_outer was neutral after cfr08: both sides null-introduce, so all 12 Utf8
value lanes gather through reindex_by_positions' null path (cmxjz) — that is NOT
the kept-lane clone cfr08 fixed; it is the byte materialization, now deferred.

SCORE: str_outer 2.70x and str_left 2.43x both >= 2.0 PASS.

ISOMORPHISM:
  - Materialized values byte-identical to the eager cmxjz gather: present slot ==
    source[pos].clone(); missing slot == Scalar::Null(NullKind::Null) ==
    missing_for_dtype(Utf8) (same as the LazyNullableUtf8 sibling + Scalar fallback).
  - Validity mask built eagerly, bit set iff position present and in range —
    identical to the cmxjz `words` mask.
  - Ordering preserved: positions consumed in output order; no reordering.
  - Gated on all-valid (self.validity.all()) Eager source, so every present
    position is a real Utf8 string and its bit is correctly set.
