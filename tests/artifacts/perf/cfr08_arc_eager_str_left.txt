br-frankenpandas-cfr08 — COW/Arc-shared ScalarValues::Eager backing (O(1) Column clone)
=========================================================================================

LEVER (one): ScalarValues::Eager(Vec<Scalar>) -> Eager(Arc<[Scalar]>).
Column::clone of a Scalar-backed (Eager) column now shares the Arc in O(1)
instead of deep-copying every Scalar (per-String malloc for Utf8). The kept
(identity) Utf8 lanes of a left join clone their value columns -> now zero-copy.
Backing is immutable after construction (ScalarValues exposes only &[Scalar]
via Deref; no &mut / IndexMut / push path; every "mutation" rebuilds the whole
`values` field), so the share is observationally a deep copy. No Arc::make_mut
needed. 6 sites, all in fp-columnar/src/lib.rs (enum, from_vec, deserialize;
as_slice/len/clone unchanged via deref coercion + Arc::clone).

BENCH (perf_profile, release-perf, n=100000, 100 iters/run, ms/iter, min-of-5):

                  bottleneck is reindex_by_positions null gather, not kept-lane clone)

ISOMORPHISM:
  - Ordering preserved: yes — Arc<[Scalar]> holds identical Scalars in identical
    order; as_slice()/Deref returns the same &[Scalar].
  - Floating-point: identical — Scalars byte-copied once at from_vec; shared reads
    return identical bytes; no value transformation.
  - Mutation safety: no &mut backing exists; share can never observe a write.
